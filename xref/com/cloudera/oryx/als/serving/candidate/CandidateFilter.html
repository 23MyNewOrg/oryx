23  /**
24   * <p>Implementations of this interface speed up the recommendation process by pre-selecting a set of items
25   * that are the only possible candidates for recommendation, or, are the most likely to be top recommendations.</p>
26   * 
27   * <p>For example, an implementation might know that most items in the model, while useful as data, 
28   * are not recommendable since they are old -- out of date, or at least, undesirable enough as recommendations 
29   * to be excluded. It could pre-compute which items are eligible and </p>
30   * 
31   * <p>This is a form of filtering, but, differs from the filtering provided by 
32   * {@link com.cloudera.oryx.als.serving.RescorerProvider}. That is a run-time, per-request filter;
33   * this class represents a more global, precomputed filtering that is not parameterized by the request.</p>
34   *
35   * <p><em>This is quite a low-level API and should be considered "advanced" usage; use a
36   * {@link com.cloudera.oryx.als.serving.RescorerProvider} unless it's clear that it is not fast enough.</em></p>
37   *
38   * <p>Implementations should define a constructor that accepts a parameter of type {@link LongObjectMap}.
39   * This is a reference to the "Y" matrix in the model -- item-feature matrix.
40   * Access to Y is protected by a lock, but, the implementation can assume that it is locked for
41   * reading (not writing) during the constructor call, and is locked for reading (not writing) during
42   * a call to {@link #getCandidateIterator(float[][])} and while the result of that method is used.
43   * So, implementations may save and use a reference to Y if it is only used in the context of these
44   * methods and only for reading.</p>
45   * 
46   * @author Sean Owen
47   * @see com.cloudera.oryx.als.serving.RescorerProvider
48   */
49  public interface CandidateFilter {
50    
51    // Note that your implementation will need a constructor matching the following, which is how it
52    // gets a reference to the set of items:
53    
54    // public YourCandidateFilter(FastByIDMap<float[]> Y) {
55    //   ...
56    // }
57  
58    /**
59     * @param userVectors user feature vector(s) for which recommendations are being made. This may or may not
60     *  influence which items are returned. Use {@link com.cloudera.oryx.als.common.StringLongMapping#toLong(String)}
61     *  to find the numeric internal ID for a given string ID.
62     * @return a set of items most likely to be a good recommendation for the given users. These are returned
63     *  as item ID / vector pairs ({@link LongObjectMap}'s {@code MapEntry}). They are returned as an {@link Iterator},
64     *  and not just one, but potentially many. If several are returned, then the caller can process the
65     *  {@link Iterator}s in parallel for speed.
66     */
67    Collection<Iterator<LongObjectMap.MapEntry<float[]>>> getCandidateIterator(float[][] userVectors);
68  
69    /**
70     * Note a new item has appeared at run-time.
71     *
72     * @param itemID ID of new item
73     */
74    void addItem(String itemID);
