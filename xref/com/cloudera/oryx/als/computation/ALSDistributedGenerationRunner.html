67  /**
68   * <p>This class will run one generation of the Computation Layer. It is essentially a client of Hadoop
69   * and starts and monitors a series of MapReduce jobs that complete the Computation Layer's work.</p>
70   *
71   * <p>This is an overview of the sequence of MapReduce jobs that can run as part of the ALS pipeline.
72   * While it is one flow, there are several distinct concerns inside of it, some optional. The core
73   * flow is presented first, followed by notes about secondary or optional aspects of the flow. These
74   * are presented in order of execution although in many cases some jobs run in parallel with others.</p>
75   *
76   * <p>In all cases, generation N's data is found at HDFS path {@code /[bucket]/[N]/} (where N is
77   * padded to 5 digits so that lexical order equals numeric order). Locations are relative to this
78   * unless otherwise specified.</p>
79   *
80   * <ul>
81   * <li>"varlong" refers to the representation {@code long}</li>
82   * <li>"varint" refers to {@code int}</li>
83   * <li>"idvalue" refers to {@code IDValue}</li>
84   * <li>"floatarray" refers to {@code float[]}</li>
85   * <li>"floatmap" refers to {@code FastByIDFloatMap}</li>
86   * <li>"idset" refers to {@code FastIDSet}</li>
87   * </ul>
88   *
89   *
90   * <h2>Core Steps</2>
91   *
92   * <h3>MergeNewOldStep</h3>
93   *
94   * <p>Merges all data from last generation with new input in the current generation. The last generation's
95   * data exists in generation N-1 under {@code input/}. It encodes user,item,value tuples as
96   * user-(item,value) key/value pairs, in particular as varlong mapped to idvalue objects.</p>
97   *
98   * <p>New input is always found in generation N under {@code inbound/}, in (compressed) CSV text files.
99   * This step merges old and new values by adding them. Here is where 'decay' of old data takes place too.</p>
100  *
101  * <p>Output goes to generation N's {@code input/}, and this is what drives all further computation in
102  * the generation.</p>
103  *
104  * <h3>ToUserVectorsStep / ToItemVectorsStep</h3>
105  *
106  * <p>These simply transform the output above into a mapping from user to all item/value pairs as a single
107  * floatmap object -- user vectors if you like -- and likewise for item vectors. The output goes to
108  * {@code userVectors/} and {@code itemVectors/} respectively.</p>
109  *
110  * <h3>CollectKnownItemsStep</h3>
111  *
112  * <p>Outputs user IDs mapped to a set of item IDs that the user ID has interacted with in the data set.
113  * The output is in text format, in {@code knownItems/} and is for consumption by the Serving Layer.</p>
114  *
115  * <h3>InitialYStep</h3>
116  *
117  * <p>This process outputs an initial Y matrix. It will optionally reuse the final {code Y/} output from
118  * last generation, N-1. For any items that have no vector yet (which might be all of them on a fresh run)
119  * it will choose random unit vectors (chosen with kmeans++-style preference for dissimilar vectors).
120  * Output goes to {@code tmp/iterations/0/Y}, and is encoded as varlong mapped to floatarray.</p>
121  *
122  * <h3>PopularUserStep / PopularItemStep</h3>
123  *
124  * <p>Slightly misnamed, as they actually output the set of items that a given user interacts with in the
125  * data, and vice versa for items. The output is encoded as varlong mapped to idset, and the output goes
126  * to {@code popularUsersByItemPartition} and {@code popularItemsByUserPartition} respectively.. This is
127  * used later to cleverly side-load only the part of the X or Y matrix that will actually be used by
128  * a reducer.</p>
129  *
130  * <h3>RowStep</h3>
131  *
132  * <p>The heart of the ALS iteration, and the most important step in the flow. This is the one that is
133  * iterated many times. It will alternately compute X from Y, then Y from X. In the first case, for
134  * iteration i, input is from {@code tmp/iterations/[i-1]/Y} and output goes to
135  * {@code tmp/iterations/[i]/X}. In the second case, it's {@code tmp/iterations/[i]/X} going to
136  * {code tmp/iterations/[i]/Y}. Input and output are rows of a matrix: varlong mapped to floatarray.</p>
137  *
138  * <p>The core ALS computation here is fairly straightforward, but, there is complexity around
139  * side-loading the X or Y matrix, rather than literally joining the matrix on the input, which would
140  * be far too slow. To conserve memory, only keys that are expected to appear in the input to the
141  * particular reducer will be loaded.</p>
142  *
143  * <h3>PublishXStep / PublishYStep</h3>
144  *
145  * <p>Simple map-only jobs that publish the X or Y matrix, respectively, from the final iteration, in a
146  * form that's consumable by the Serving Layer. The output is compressed text format. See the source
147  * code for exact format. It is otherwise the same matrix as computed by the iterations. The output
148  * goes to {@code X/} and {@code Y/} respectively.</p>
149  *
150  * <h2>Recommend Steps</h2>
151  *
152  * <h3>DistributeRecommendWorkStep</h3>
153  *
154  * <p>This first step effectively joins data from {@code X/} (user-feature matrix) and {@code knownItems/},
155  * since both are needed to recommend for a user. It writes the combined values as varlong mapped to
156  * {@code IDAndKnownItemsAndFloatArrayWritable}. The output goes to {@code tmp/distributeRecommend/}.
157  * This is necessary as its own step because the shuffle/combiner will need to be used differently in
158  * the next step.</p>
159  *
160  * <h3>RecommendStep</h3>
161  *
162  * <p>This computes recommendations given output above. The item-features matrix, Y, is also needed.
163  * It is side-loaded cleverly as in {@code RowStep}. Each reducer makes N recommendations per user when
164  * N recommendations are desired. They will be condensed into N overall in the next step. The output
165  * goes to {@code tmp/partialRecommend} and is encoded as varlong (user ID) mapped to a series of idvalue
166  * objects representing top recommended items.</p>
167  *
168  * <h3>CollectRecommendStep</h3>
169  *
170  * <p>This stage simply takes output above and outputs final N recommendations for each user. Output
171  * is conceptually similar, but in text format, and goes to {@code recommend/}.
172  *
173  *
174  * <h2>Similarity Steps</h2>
175  *
176  * <h3>DistributeSimilarWorkStep</h3>
177  *
178  * <p>Quite analogous to {@code DistributeRecommendWorkStep} and {@code RecommendStep} combined. Because
179  * there is no join needed, these two tasks can be accomplished in a single job.</p>
180  *
181  * <h3>SimilarStep</h3>
182  *
183  * <p>Quite analogous to {@code CollectRecommendStep}.</p>
184 
185  * @author Sean Owen
186  */

191   @Override
192   protected List<DependsOn<Class<? extends JobStep>>> getPreDependencies() {
193     List<DependsOn<Class<? extends JobStep>>> preDeps = Lists.newArrayList();
194     preDeps.add(DependsOn.<Class<? extends JobStep>>first(MergeIDMappingStep.class));
195     preDeps.add(DependsOn.<Class<? extends JobStep>>nextAfterFirst(ToUserVectorsStep.class, MergeNewOldStep.class));
196     preDeps.add(DependsOn.<Class<? extends JobStep>>nextAfterFirst(ToItemVectorsStep.class, MergeNewOldStep.class));
197     preDeps.add(DependsOn.<Class<? extends JobStep>>nextAfterFirst(CollectKnownItemsStep.class, ToUserVectorsStep.class));
198     preDeps.add(DependsOn.<Class<? extends JobStep>>nextAfterFirst(InitialYStep.class, ToItemVectorsStep.class));
199     preDeps.add(DependsOn.<Class<? extends JobStep>>nextAfterFirst(PopularUserStep.class, ToUserVectorsStep.class));
200     preDeps.add(DependsOn.<Class<? extends JobStep>>nextAfterFirst(PopularItemStep.class, ToItemVectorsStep.class));
201     return preDeps;
202   }

220       postDeps.add(DependsOn.<Class<? extends JobStep>>nextAfterFirst(CollectRecommendStep.class, RecommendStep.class));
221     }
222     if (config.getBoolean("model.item-similarity.compute")) {
223       postDeps.add(DependsOn.<Class<? extends JobStep>>nextAfterFirst(SimilarStep.class, DistributeSimilarWorkStep.class));
224     }
225     return postDeps;

252   @Override
253   protected boolean areIterationsDone(int iterationNumber) throws IOException {
254     if (iterationNumber < 2) {
255       return false;
256     }
257 
258     Config config = ConfigUtils.getDefaultConfig();
259     int maxIterations = config.getInt("model.iterations.max");
260 
261     if (maxIterations > 0 && iterationNumber >= maxIterations) {
262       log.info("Reached iteration limit");
263       return true;
264     }
265 
266     String iterationsPrefix = Namespaces.getIterationsPrefix(getInstanceDir(), getGenerationID());
267 
268     String previousConvergenceKey = iterationsPrefix + (iterationNumber-1) + "/Yconvergence/";
269     String currentConvergenceKey = iterationsPrefix + iterationNumber + "/Yconvergence/";
270     Store store = Store.get();
271     Preconditions.checkState(store.exists(currentConvergenceKey, false),
272                              "No convergence samples in current iteration");
273     if (!store.exists(previousConvergenceKey, false)) {
274       // Previous iteration was deleted already because next iteration failed to start; continue
275       return false;
276     }
277 
278     LongObjectMap<LongFloatMap> previousEstimates = readUserItemEstimates(previousConvergenceKey);
279     LongObjectMap<LongFloatMap> estimates = readUserItemEstimates(currentConvergenceKey);
280     Preconditions.checkState(estimates.size() == previousEstimates.size(),
281                              "Estimates and previous estimates not the same size: %s vs %s",
282                              estimates.size(), previousEstimates.size());
283 
284     DoubleWeightedMean averageAbsoluteEstimateDiff = new DoubleWeightedMean();
285     for (LongObjectMap.MapEntry<LongFloatMap> entry : estimates.entrySet()) {
286       long userID = entry.getKey();
287       LongFloatMap itemEstimates = entry.getValue();
288       LongFloatMap previousItemEstimates = previousEstimates.get(userID);
289       Preconditions.checkState(itemEstimates.size() == previousItemEstimates.size(),
290                                "Number of estaimtes doesn't match previous: {} vs {}",
291                                itemEstimates.size(), previousItemEstimates.size());
292       for (LongFloatMap.MapEntry entry2 : itemEstimates.entrySet()) {
293         long itemID = entry2.getKey();
294         float estimate = entry2.getValue();
295         float previousEstimate = previousItemEstimates.get(itemID);
296         // Weight, simplistically, by newValue to emphasize effect of good recommendations.
297         // But that only makes sense where newValue > 0
298         if (estimate > 0.0f) {
299           averageAbsoluteEstimateDiff.increment(FastMath.abs(estimate - previousEstimate), estimate);
300         }
301       }
302     }
303 
304     double convergenceValue;
305     if (averageAbsoluteEstimateDiff.getN() == 0) {
306       // Fake value to cover corner case
307       convergenceValue = FastMath.pow(2.0, -(iterationNumber + 1));
308       log.info("No samples for convergence; using artificial convergence value: {}", convergenceValue);
309     } else {
310       convergenceValue = averageAbsoluteEstimateDiff.getResult();
311       log.info("Avg absolute difference in estimate vs prior iteration: {}", convergenceValue);
312       if (!Doubles.isFinite(convergenceValue)) {
313         log.warn("Invalid convergence value, aborting iteration! {}", convergenceValue);
314         return true;
315       }
316     }
317 
318     double convergenceThreshold = config.getDouble("model.iterations.convergence-threshold");
319     if (convergenceValue < convergenceThreshold) {
320       log.info("Converged");
321       return true;
322     }
323     return false;
324   }

326   private static LongObjectMap<LongFloatMap> readUserItemEstimates(String convergenceSamplePrefix)
327       throws IOException {
328     log.info("Reading estimates from {}", convergenceSamplePrefix);
329     LongObjectMap<LongFloatMap> userItemEstimate = new LongObjectMap<LongFloatMap>();
330     Store store = Store.get();
331     for (String prefix : store.list(convergenceSamplePrefix, true)) {
332       for (CharSequence line : new FileLineIterable(store.readFrom(prefix))) {
333         String[] tokens = DelimitedDataUtils.decode(line);
334         long userID = Long.parseLong(tokens[0]);
335         long itemID = Long.parseLong(tokens[1]);
336         float estimate = Float.parseFloat(tokens[2]);
337         LongFloatMap itemEstimate = userItemEstimate.get(userID);
338         if (itemEstimate == null) {
339           itemEstimate = new LongFloatMap();
340           userItemEstimate.put(userID, itemEstimate);
341         }
342         itemEstimate.put(itemID, estimate);
343       }
344     }
345     return userItemEstimate;
346   }

348   @Override
349   protected void doPost() throws IOException {
350     log.info("Loading X and Y to test whether they have sufficient rank");
351 
352     String generationPrefix = Namespaces.getInstanceGenerationPrefix(getInstanceDir(), getGenerationID());
353     String xPrefix = generationPrefix + "X/";
354     String yPrefix = generationPrefix + "Y/";
355 
356     Store store = Store.get();
357     if (!doesXorYHasSufficientRank(xPrefix) || !doesXorYHasSufficientRank(yPrefix)) {
358       log.warn("X or Y does not have sufficient rank; deleting this model and its results");
359       store.recursiveDelete(xPrefix);
360       store.recursiveDelete(yPrefix);
361       store.recursiveDelete(generationPrefix + "recommend/");
362       store.recursiveDelete(generationPrefix + "similarItems/");
363     } else {
364       log.info("X and Y have sufficient rank");
365       File tempModelDescriptionFile = File.createTempFile("model-", ".pmml.gz");
366       tempModelDescriptionFile.deleteOnExit();
367       ALSModelDescription modelDescription = new ALSModelDescription();
368       modelDescription.setKnownItemsPath("knownItems");
369       modelDescription.setXPath("X");
370       modelDescription.setYPath("Y");
371       modelDescription.setIDMappingPath("idMapping");
372       ALSModelDescription.write(tempModelDescriptionFile, modelDescription);
373       store.upload(generationPrefix + "model.pmml.gz", tempModelDescriptionFile, false);
374       IOUtils.deleteRecursively(tempModelDescriptionFile);
375     }
376 
377     super.doPost();
378   }

380   private static boolean doesXorYHasSufficientRank(String xOrYPrefix) throws IOException {
381     double[][] transposeTimesSelf = null;
382     Store store = Store.get();
383     for (String xOrYFilePrefix : store.list(xOrYPrefix, true)) {
384       for (String line : new FileLineIterable(store.readFrom(xOrYFilePrefix))) {
385         int tab = line.indexOf('\t');
386 
387         float[] elements = DataUtils.readFeatureVector(line.substring(tab + 1));
388         int features = elements.length;
389 
390         if (transposeTimesSelf == null) {
391           transposeTimesSelf = new double[features][features];
392         }
393 
394         for (int i = 0; i < features; i++) {
395           double rowFactor = elements[i];
396           double[] transposeTimesSelfRow = transposeTimesSelf[i];
397           for (int j = 0; j < features; j++) {
398             transposeTimesSelfRow[j] += rowFactor * elements[j];
399           }
400         }
401 
402       }
403       
404       // Hacky but efficient and principled: if a portion of the matrix times itself is non-singular,
405       // it's all but impossible that it would be singular when the rest was loaded
406       
407       if (transposeTimesSelf != null && 
408           transposeTimesSelf.length != 0 && 
409           MatrixUtils.isNonSingular(new Array2DRowRealMatrix(transposeTimesSelf))) {
410         return true;
411       } else {
412         log.info("Matrix is not yet proved to be non-singular, continuing to load...");
413       }
414     }
415     return false;
416   }
