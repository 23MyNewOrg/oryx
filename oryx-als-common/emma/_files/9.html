36/**
37 * <p>This class implements a form of location sensitive hashing (LSH). This is used to quickly, approximately,
38 * find the vectors in the same direction as a given vector in a vector space. This is useful in, for example, making
39 * recommendations, where the best recommendations are the item vectors with largest dot product with
40 * the user vector. And, in turn, the largest dot products are found from vectors that point in the same direction
41 * from the origin as the user vector -- small angle between them.</p>
42 *
43 * <p>This uses H hash functions, where the hash function is based on a short vector in a random direction in
44 * the space. It suffices to choose a vector whose elements are, randomly, -1 or 1. This is represented as a
45 * {@code boolean[]}. The vector defines a hyperplane through the origin, and produces a hash value of 1 or 0
46 * depending on whether the given vector is on one side of the hyperplane or the other. This amounts to
47 * evaluating whether the dot product of the random vector and given vector is positive or not.</p>
48 *
49 * <p>These H 1/0 hash values are combined into a signature of H bits, which are represented as an {@code long}
50 * because for purposes here, H is less than or equal to 64.</p>
51 *
52 * <p>"Close" vectors -- those which form small angles together -- point in nearly the same direction and so
53 * should generally fall on the same sides of these hyperplanes. That is, they should match in most bits.</p>
54 *
55 * <p>As a preprocessing step, all item vector signatures are computed, and these define a sort of
56 * hash bucket key for item vectors. Item vectors are put into their buckets.</p>
57 *
58 * <p>To produce a list of candidate item vectors for a given user vector, the user vector's signature is
59 * computed. All buckets whose signature matches in "most" bits are matches, and all item vectors inside
60 * are candidates.</p>
61 *
62 * <p><em>This is experimental, and is disabled unless "model.lsh.sample-ratio" is set to a value less than 1.</em></p>
